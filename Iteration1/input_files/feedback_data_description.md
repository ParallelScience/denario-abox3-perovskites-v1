The observed limitations in the analysis are directly attributable to constraints described in the dataset:

1. **Elastic data sparsity**: The dataset description explicitly notes that only 215/1283 (17%) of materials have computed elastic constants. This constraint necessitated the use of a two-stage pipeline and the exclusion of 1068 materials from the initial training of the mechanical models, limiting the direct empirical basis for mechanical property predictions.
2. **Measurement noise/DFT artifacts**: The description acknowledges "pathological outliers" and "unphysical noise" in the elastic data (e.g., negative moduli). This constraint forced the research plan to implement strict physical bounds (0 < K_VRH < 300 GPa) and quantile regression to mitigate the impact of these artifacts on model training.
3. **Theoretical nature of data**: The dataset consists entirely of DFT-calculated properties with no experimental validation. This constraint limits the conclusions to "predicted" viability and "surrogate" performance, as the models are inherently bounded by the systematic errors of the PBE functional (e.g., band gap underestimation) mentioned in the caveats.
